4:00 AM ET
Pre-Market Opens

You Start the App

Market Opens

12:00-2:00 PM
Midday Lull

2:00-4:00 PM
Power Hour
Volume picks up, EOD setups, position sizing for overnight holds.
4:00 PM ET
Market Closes
Final position sync, trade journal updated, learning connections sync, daily P&L calculated.

AI Decision Making

When an alert comes in, SentCom runs through a multi-step analysis process before deciding to trade.

1

Initial Filtering

ADV check (500K+ volume), price range, spread check, blacklist check

Pass → Continue | Fail → Reject
2

Intelligence Gathering

Fetches news sentiment, technical indicators (RSI, MACD, EMAs), and setup quality metrics in parallel.

3

Bull vs Bear Debate

Two AI agents argue for and against the trade. Each presents arguments with scores.

🐂 Bull Agent

Pattern quality, volume, catalyst, technicals

🐻 Bear Agent

Resistance, overbought, sector weakness, risks

4

Risk Assessment

Evaluates position size risk, market regime, correlation, volatility, liquidity, time of day, and historical performance.

5

Trade Construction

Calculates entry, stop loss, position size (risk-based), and scale-out targets.

6

Execution

In autonomous mode, executes automatically. In manual mode, shows alert for your confirmation.

Trade Execution Flow

📤 Order Flow
Cloud Backend → IB Data Pusher → IB Gateway → Exchange → FILL ↓ ↓ Trade Journal ← Position Monitor ← Stop Loss Placed ← ──┘
📝

Order Types

Limit orders for entries, stop-loss orders for protection, scale-out orders for targets.

🎯

Scale-Out Strategy

Sell 33% at Target 1, 33% at Target 2, remainder at Target 3 or trailing stop.

🛑

Stop Management

Initial stop set on entry. After T1 hit, stop moves to breakeven. After T2, locks in profit.

📊

Position Monitoring

Real-time P&L updates, target tracking, and optional trailing stop activation.

⚠️
Important
IB Gateway must be running and connected for order execution. If disconnected, orders are simulated only.

Historical Data Collection

SentCom collects historical OHLCV data from IB Gateway to train AI models and analyze patterns.

🗂️ Data Tiers by Volume
Tier ADV Range Symbols Timeframes Collected
Intraday 500K+ shares/day ~2,500 1min, 5min, 15min, 30min, 1hr, 1day
Swing 100K-500K ~1,400 5min, 30min, 1hr, 1day
Investment 50K-100K ~700 1hr, 1day, 1week
🚀 Running the Historical Collector
# Start collection with optimized settings .\documents\scripts\StartHistoricalCollector.bat --fast # Options: --fast 8 batch size, 0.5s delay (recommended) --turbo 10 batch size, 0.3s delay (may hit limits) --slow 2 batch size, 2s delay (conservative) --skip-login Skip IB Gateway restart if already running
⚡
Smart Skip Feature
The collector automatically skips symbols that already have complete data, checking timeframe-specific thresholds (e.g., 1500+ bars for daily data). This significantly speeds up gap-filling runs.

Troubleshooting

🔧 Common Issues
IB Gateway won't connect

• Check IB Gateway is running and logged in
• Verify port 4002 is not blocked
• Ensure API connections are enabled in TWS/Gateway settings

Client ID already in use

• Close other applications using IB Gateway
• Change client ID in script (99 for collector, 1 for backend)

Backend not responding

• Check backend window for errors
• Run: curl http://localhost:8001/api/health
• Restart backend if needed

Pacing violations (IB limit)

• Normal - collector auto-waits when hitting limits
• Use --slow mode if hitting too many limits
• IB allows ~60 requests per 10 minutes
