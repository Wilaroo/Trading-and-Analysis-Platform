The Trading Day

Timeline of what happens throughout a typical trading day.

4:00 AM ET
Pre-Market Opens
Scanner begins monitoring pre-market movers, gap scanner identifies overnight gaps, news service fetches overnight headlines.
8:00 AM ET
You Start the App
StartTrading.bat launches, IB Gateway connects, system syncs open positions.
9:30 AM ET
Market Opens
Scanner shifts to intraday mode, ORB (Opening Range Breakout) setups activate, first 15-30 mins: high volatility scanning.
9:30-10:00 AM
ORB Window
Opening range established, ORB breakout alerts generated. Example: "NVDA ORB Breakout Long at $485.50"
10:00-12:00 PM
Morning Session
VWAP reclaim/reject plays, 9 EMA scalps, pattern breakouts (flags, triangles).
12:00-2:00 PM
Midday Lull
Scanner widens thresholds (less activity), focus on swing setups, daily chart patterns.
2:00-4:00 PM
Power Hour
Volume picks up, EOD (End of Day) setups, position sizing for overnight holds.
4:00 PM ET
Market Closes
Final position sync with IB, trade journal updated, learning connections sync, daily performance calculated.

Data Collection

What data SentCom collects and stores for trading and learning.

📊 Real-Time Data Streams

📈

Price Data

Bid/Ask/Last every 250ms, Volume tick-by-tick, High/Low of day from IB Gateway

📰

News Data

IB News feed (primary), Finnhub headlines (backup), Sentiment analysis on each

🌍

Market Context

SPY/QQQ levels, VIX level, Sector performance, Market breadth

📦 Historical Data Collection Flow

Cloud Creates Request

Request queued: AAPL, 1 month, daily bars → Status: "pending"

IB Data Pusher Polls

GET /api/ib/historical-data/pending → Sees AAPL request → Claims it (status → "claimed")

Fetch from IB Gateway

Sends request to IB Gateway (port 4002) → IB fetches from servers → Returns ~22 daily bars

Report Back to Cloud

POST /api/ib/historical-data/result → Data stored in ib_historical_data collection

9

End of Day

What happens when the market closes and you shut down.

Final Position Sync (4:00 PM)

Compare SentCom positions vs IB Gateway, flag any discrepancies, update P&L with closing prices

Trade Journal Update

Save all closed trades to database, calculate final R-multiples, tag each trade with setup type, time, outcome

Learning Sync

Update alert_outcomes with today's results, run scanner calibration if needed, update module weights if accuracy changed

Daily Stats Calculation

Total P&L, trade count, win rate, best/worst trades - all logged for your records

Prediction Verification

Check Time-Series AI predictions vs actual moves, log accuracy for model improvement

Cleanup

Clear expired alerts, archive old queue items, compress logs

🔌 Shutting Down

IB Data Pusher: Disconnects from cloud (pending requests stay in queue for next session)
IB Gateway: Closes or stays open based on settings (Auto-Logoff at 11:59 PM, Auto-Restart at 1:45 AM)
Cloud Backend: Keeps running - all data safe, learning continues, ready for next session
