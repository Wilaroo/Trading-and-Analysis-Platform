**Key Technical Concepts**
-   **Dual-Driver DB Abstraction**: The  was refactored with a  helper method that detects whether it was given a  (sync) or  (async) DB object and calls the database with the correct pattern ( or ), making the component reusable in both the server and worker processes.
-   **Background Worker Architecture**: The session solidified the architecture of using a separate  process for all heavy tasks (training, data collection, etc.) via a MongoDB job queue, keeping the main FastAPI server responsive.
-   **Request Throttling**: Identified that a request throttler with too low a limit was causing UI freezes and silent failures by queuing critical POST requests behind dozens of polling GET requests. The solution was to use a non-throttled API instance for long-running or critical actions.
-   **UI State Recovery**: Implemented logic in the  component to check for running jobs on mount, allowing the UI to reconnect and display progress for a long-running job even after a page refresh.
