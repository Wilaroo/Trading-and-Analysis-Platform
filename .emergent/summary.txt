**Key Technical Concepts**
-   **Cloud-to-Local Command Queue**: The new core architecture for data collection. The cloud backend is decoupled from the local data source (IB Gateway). The cloud acts as a job coordinator, and the local  script acts as a distributed worker. This is a critical pattern to understand.
-   **Asynchronous Job Processing**: Jobs are created, polled, executed, and reported on asynchronously, making the system resilient to network issues and long-running tasks.
-   **Dynamic Data Filtering (ADV & IB Compatibility)**: The system now uses multiple layers of filtering. First, it uses a pre-computed ADV cache to select a liquid universe of stocks. Second, it applies a regex-based filter to remove symbols known to be incompatible with IB Gateway before a job even starts.
